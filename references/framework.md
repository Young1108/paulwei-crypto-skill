# Paul Wei 交易框架核心原则参考

> 基于 173,058笔成交 / 6年公开实盘数据逆向工程
> 详细原则以本文件摘要和 README 中列出的公开数据来源为准

> **非 BTC 品种说明**：本框架所有阈值（MA30±10%、杠杆上限等）均来自 BTC 历史数据的实证观察。用于 ETH/SOL 等其他币种时，方向和逻辑适用，具体数值为合理推断而非实证结论。山寨币波动率更高，实际偏离阈值可适当放宽（例如 MA30±15%）。

## 一、仓位管理原则

### 1.1 仓位大小是核心风险控制手段

Paul Wei 基本不用止损单（占总成交<1%），但这不代表可以忽略退出条件。框架的核心是**把每笔仓位做得足够小**，并同时明确结构失效条件和退出计划。

```
单笔最大亏损 = 账户净值 × 0.5%（日常）
             = 账户净值 × 3%（极限，三维信号触发）

连续错10次的最大损失：
  0.5% × 10 = 5%（账户仍可正常运转）
```

### 1.2 仓位随账户成长同比例扩张

他不用固定美元金额，而是**账户净值的百分比**定义仓位。这使仓位随账户自动复利扩张。

### 1.3 建仓慢，退出快

| 阶段 | 中位时长 | 方式 |
|---|---|---|
| 建仓 | 3.5小时 | Maker挂单为主 |
| 退出 | 2.4小时 | 信号出现后较快出清 |
| 建仓分散度 | 0.46% | 刻意分散价格区间 |
| 退出分散度 | 0.29% | 相对集中快速出清 |

## 二、方向判断原则

### 2.1 三个确认维度

真正的入场信号需要至少两个维度同时确认：
1. **BTC/NASDAQ相对强弱**（TradingView手动确认）
2. **价格结构**（MA30偏离 + 支撑阻力位置）
3. **宏观事件状态**（近期有无重大事件）

### 2.2 均线偏离状态

| 偏离MA30 | 历史仓位倾向 | 操作含义 |
|---|---|---|
| > +10% | **偏空/减多** | 过热，做空波段或减仓 |
| +5~10% | 中性偏谨慎 | 不追多，等回调 |
| -2~+2% | **偏多（最强信号）** | 均线支撑区，多头建仓区 |
| -5~-10% | 中性偏谨慎 | 不追空，等反弹 |
| < -10% | **偏多（抄底）** | 超跌，逆势建多 |

### 2.3 宏观事件处理

| 事件类型 | 处理方式 |
|---|---|
| 已知利好即将兑现 | 减仓等待，兑现后可能做反向 |
| 已知利空即将兑现 | 同上，不追跌 |
| 极端不确定事件 | 空仓，关机不交易 |
| 利好/利空已兑现 | 观察价格反应，"卖现实"机会 |

他系统性地在利好兑现时做空（ETF通过/特朗普就职/首次降息均持空仓）。这不是猜新闻，而是读"消息被定价的程度"。

### 2.4 相对强弱解读

**BTC（主要工具）**：
```
TradingView自定义比率：BTCUSD / NASDAQ

比率上涨 + BTC/USD上涨 = 真实强势 → 偏多
比率上涨 + BTC/USD下跌 = 假弱实强 → 逆势多
比率下跌 + BTC/USD上涨 = 假强实弱 → 逆势空
比率下跌 + BTC/USD下跌 = 真实弱势 → 偏空
```

**其他币种（ETH/SOL 等）**：
```
TradingView比率：{币种}/BTC（如 ETHBTC、SOLBTC）

比率上涨 = 该币强于BTC → 做多偏向该币
比率下跌 = 该币弱于BTC → 谨慎做多，BTC更优选

注意：做多山寨币时，若ETH/BTC或SOL/BTC同步下跌，
该涨势可能只是BTC带动，并非该币内生强势。
```

**本 skill 的限制**：工具不直接获取 NASDAQ 数据，相对强弱需用户在 TradingView 手动确认。

## 三、进场执行原则

### 3.1 挂单心理整数位

```
❌ 避免：整千整百（$75,000 / $74,000）→ 排队最多
✅ 推荐：末3位重复数字
  多单: $74,999 / $74,888 / $74,777 / $74,666
  空单: $75,111 / $75,222 / $75,333 / $75,444
挂单距现价：不超过价格的 0.05-0.15%（BTC约$50-150，SOL约$0.05-0.15）
```

### 3.2 三维爆发信号识别

当以下三个维度**同时**切换，表示他确信了方向：

| 维度 | 日常状态 | 爆发状态 |
|---|---|---|
| 订单类型 | Maker挂单 | **Taker吃单** |
| 操作间隔 | 数小时 | **5-15秒** |
| 单笔数量 | 基准量 | **4-60倍基准量** |

只有三个维度同时切换才是真正的重拳。

### 3.3 反弹/回调是加仓机会

- 做多期间回调 → 挂单自动加多（更低成本）
- 做空期间反弹 → 挂单自动加空（更高成本）
- 浮亏时加仓量是浮盈时的1.69倍

**前提**：单笔极小（0.5%净值），错10次才亏5%。

## 四、止盈原则

### 4.1 利润捕获率目标：84% MFE

不追求最高点出场，在动能减弱时开始分批减仓：
- 每走$3-5K USD，机械减仓一部分
- 49%的减仓在浮盈过程中完成（不等回头）
- 宁可少赚最后一段，也不博顶部

### 4.2 出场触发信号

- 价格快速拉升后横盘（动能减弱）
- 宏观利好事件兑现（"卖现实"）
- BTC/NASDAQ出现背离（价格新高但比率未创新高）
- 偏离MA30超过+15%（过热极值）

## 五、风险管理红线

以下行为在他的体系中不存在，是交易失败的主要原因：

| 行为 | 原因 |
|---|---|
| 全仓或接近全仓 | 一次失误即灾难性损失 |
| 实际杠杆 > 5x | 超过账户可承受范围 |
| 连续亏损后立刻加大仓位 | 典型报复性交易，情绪驱动 |
| 追涨杀跌（价格创新高追多）| 最差的入场时机 |
| 大事件前押大注 | 二元事件风险不可控 |

## 七、情绪识别快速参考

### 贪婪指纹（危险信号）
- 账户处于近期高点，仓位却大幅扩张
- 切换为Taker快速建仓（放弃Maker耐心）
- 仓位超过正常水平50%以上

### 恐惧指纹（危险信号）
- 大亏损后，小仓位高频试错
- 多空方向快速切换，无明确判断
- 连续多日亏损仍继续交易（停不下来）

### 正确状态特征
- Maker挂单为主（等价格来）
- 21.7%以上的时间空仓不交易
- 每月交易次数 ≤ 15-20次
- 最大回撤不超过账户5%
